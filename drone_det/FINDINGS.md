# 안티드론 합성데이터 — 검증 결과 종합 (정직한 증거 기반)

RGB 단일 합성 파이프라인(Isaac Sim, 시부야+HDRI+자동GT+도메인랜덤화)이 실제 드론탐지에
**언제·얼마나** 가치 있는지를 2개 독립 실벤치 + 다중시드로 검증.

## ★ 마스터 발견 (results/master_when_synth_helps.png)

**합성데이터의 가치는 절대적이지 않고 `real-only 충분성`에 조건부다.**
2벤치 6개 작동점에서 `real-only mAP` vs `합성효과(B-A)` 상관 = **-0.90**.

- **real-only가 약함(난이도 높음/실라벨 부족, mAP<~0.5)** → 합성이 **크게 도움** (벤치1: 0.20→0.52)
- **real-only가 강함(쉬움/실라벨 충분, mAP>~0.5)** → 합성은 **방법 무관 해침** (벤치2: 0.73→0.17, 도메인갭)
- crossover ≈ real-only mAP 0.5

## 시뮬 현실화(Cycle6)가 한 것

진단(실드론 vs 합성 시각비교)으로 갭 3개 발견 → 수정:
- 배경: 도시지배 → 맑은 하늘 3/4 (도시 화면밖)
- 색감: 노을·과노출 → 파란 주광 HDRI + 노출보정
- 스케일: tiny만 → 중간/근접 드론 포함
- 노이즈↓ (벤치는 깔끔)

효과: 개선합성 > old합성 **두 벤치 모두**(일반화) + **분산 3~4배↓**(scarce real 안정화).
단, 합성을 "필요없는 영역(real 충분)"서 유용하게 만들진 못함 — 그건 도메인갭의 근본한계.

## 프로젝트 함의 (언제 우리 합성을 쓰나)

- ✅ **실라벨이 부족하고 태스크가 어려운** 안티드론 시나리오(다양 기종·원거리·희귀 실데이터)
  → 합성 pretrain→실FT로 mAP 크게↑ + 학습 안정화. (과제의 실제 사용처)
- ❌ 실데이터가 이미 충분/쉬운 셋업 → 합성 불필요(오히려 해침). 합성을 무조건 더하지 말 것.
- ⚠️ naive mix는 합성이 비율지배하면 붕괴 → **pretrain→실FT** 또는 real:합성 비율관리 필수.

## 한계 / 정직한 caveat

- 벤치2는 "쉬운" 벤치(real-only가 N10서 0.73)라 합성가치 검증엔 부적합했음 — 그래도 "개선>old·분산↓"는 확인.
- 벤치1 'drone'은 헬리콥터 대량 포함(라벨 특성) — 우리 합성엔 실헬기 없음(잔여에러 60%). 소형 드론은 잘 탐지.
- C(pretrain→FT)는 시드분산≈0(강한 pretrain init + 소량 real → 결정적 수렴). 비교는 유효.
