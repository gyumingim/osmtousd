# DATA_SPEC — 시뮬(AI-2) → 탐지학습(AI-1) 데이터 계약

> AI-2가 Isaac Sim/Replicator로 데이터를 뽑을 때 **이 포맷·내용대로** 주면
> AI-1의 `train.py`에 그대로 꽂힙니다. (방법론 근거 = `METHODOLOGY.md`)

---

## 1. 출력 포맷 (YOLO — 우리 기존과 동일)
```
synth_dataset/
├── images/{train,val}/*.jpg        RGB, 권장 1280x720 이상(초소형 보존)
├── labels/{train,val}/*.txt        프레임당 YOLO 라벨(아래)
├── meta/{train,val}/*.json         프레임당 메타(거리·자세·DR파라미터)  ← 추가
└── data.yaml                       nc·names·경로
```
- 라벨 1줄 = `class xc yc w h` (정규화 0~1). **class: drone=0** (다기종이면 0 통일 + meta에 기종)
- **빈 라벨 파일 = 배경(드론0) 프레임** → 반드시 일정 비율 포함(오탐↓)

## 2. 프레임 메타(`meta/*.json`) — 거리층화·자세 평가용 (필수)
```json
{
  "frame": "train_00001",
  "objects": [{
    "class": "drone", "model": "quadX",
    "bbox_px": [x,y,w,h],
    "distance_m": 137.0,           // 카메라~드론 (10~500m)
    "bbox_min_px": 14,             // 박스 최소변 픽셀
    "pose_valid": false,           // bbox_min_px<20 이면 false (자세 학습 끔)
    "pose_3d": {"yaw":.., "pitch":.., "roll":..},   // sim GT (거리무관 완벽)
    "keypoints": [[x,y,vis], ...]  // 있으면
  }],
  "background": "sky_cloudy", "lighting": "dusk",
  "motion_blur": 0.4, "noise": 0.02     // 적용된 센서사실화 값
}
```
> **핵심: 자세/키포인트는 sim이 거리무관 완벽 GT로 다 주되, `pose_valid` 플래그로
> 작은 드론은 학습에서 제외.** (500m 자세 문제 해결 — METHODOLOGY 2-4)

## 3. 생성 *내용* 주문 (방법론 기반 — 무엇을 다양화·포함)
| 항목 | 요구 | 이유 |
|---|---|---|
| **배경** | 하늘·구름·산·바다·도심 ≥5종, 골고루 | 오탐 최대원인 (DR 1순위) |
| **거리** | 10~500m 연속 분포 (특히 **<32px 구간 충분**) | 우리 데이터 38.6%가 small |
| **드론 기종** | ≥3종 (쿼드·고정익·테일시터) | PDF 기종 3종+ |
| **드론 외형** | 색·텍스처 랜덤 | 외형 불변성 |
| **센서 사실화** | **모션블러**·노이즈·압축 | 초소형 민감 + frame-dynamics 전이 |
| **방해물** | 새·비행기 일부 포함 | 드론 vs 새 오탐 |
| **음성 프레임** | 드론0 배경만 일정 비율(예 10~20%) | 오탐 억제 |
| **조명/날씨** | 주간·황혼·야간·안개 | 환경 3종+ |

## 4. ⭐ 시계열용 — 영상(시퀀스) 구조 필수
frame-dynamics(광류·프레임차분)와 추적 학습엔 **연속 프레임**이 필요:
```
images/train/seq0007/frame_0000.jpg, frame_0001.jpg, ...   (한 비행궤적 = 한 시퀀스)
```
- 한 시퀀스 = **물리적으로 일관된 비행 궤적**(PX4/스크립트). 매 프레임 순간이동 ❌
- 메타에 `seq_id`, `frame_idx`, 드론 `velocity` 포함 → 궤적 라벨

## 5. 우선순위 (1차 = MVP)
1. **드론 1종 + 하늘 배경 몇 개 + 거리·조명 랜덤 + 2D박스 + 메타(거리/pose_valid)** — 단일 프레임 500~1000장
2. AI-1이 받아 학습·진단 → "어디서 틀리나" → 표적 주문 (반복 루프)
3. 그 다음 시퀀스(영상)·기종확장·방해물·1만장

- 좌표: YOLO 정규화 / 클래스: drone=0 / 이미지: jpg
- 거리·pose_valid·seq_id 메타 **필수**
- 이 스펙 변경은 `DATA_SPEC.md`에 기록 후 AI-1·AI-2 동기화
