# METHODOLOGY — Sim-to-Real 드론 탐지 방법론

> 역할: AI-1(나) = **방법론 설계·검증**. AI-2 = 시뮬 생성, AI-3 = 자료조사.
> 이 문서 = "어떻게 하면 Sim-to-Real + 드론 인식을 잘하나"의 권장안 + 근거.
> 근거는 ① 우리 데이터 실측 ② 검증된 최신 논문(2024~2025 안티드론 대회).

---

## 0. 한 줄 방법론
> **SOTA 레시피(YOLO+P2+1280 + 광류·프레임차분 + DR + 실데이터 fine-tune)를 깔고,
> "실패 진단 → 표적 sim 생성 → 재학습"의 반복 루프로 Sim-to-Real 갭을 좁힌다.**

---

## 1. 문제 정의 (우리 데이터로 정량화)
베이스라인(YOLO11n, val 400장) 학습 데이터 박스 2468개 분석:
| 크기(최소변) | 비율 |
|---|---|
| 극초소형 <10px | 5.2% |
| 초소형 10~20px | 19.9% |
| 소형 20~32px | 19.9% |
| 중대형 ≥32px | 55.0% |

- **COCO 'small'(<32²) = 38.6%**, **<20px = 25%**, **<16px(stride 한계권) = 13%**
- 추가로 거리 10~500m 요구 → 스케일 변동 극심.

### 1-b. 실측 진단 (CPU, best.pt 640) — 가정 검증 결과 ⚠️
크기별 recall(IoU>0.5, conf0.25):
| 크기 | recall |
|---|---|
| <16px | 0.773 (51/66) |
| 16~32px | 0.770 (87/113) |
| 32~64px | **0.711 (64/90) ← 최저** |
| ≥64px | 0.853 (139/163) |

**중요(정직): "초소형이 최악"이라는 처음 가정은 *이 데이터에선 틀림*.** recall이 0.71~0.85로
비교적 평탄하고, 오히려 32~64px가 최저. 초소형(<16px)도 0.77로 비슷.

**해석:**
1. 이 데이터셋(640×480 유튜브 프레임)은 **진짜 초소형/원거리 regime을 충분히 담지 못함**
   — 4K@500m의 <8px 케이스가 거의 없음. 그래서 tiny 문제가 *덜* 드러남.
2. → **결론이 강해짐**: *우리 실데이터로는 tiny 문제를 진단조차 제대로 못 한다*
   = **sim이 10~500m 통제된 초소형 케이스를 생성해야 하는 직접 근거.**
3. 32~64px 최저는 표본(90개) 편차일 수 있음 → 1280 실험·더 큰 val로 재확인 필요.

## 2. 정적 레시피 (어떤 기술)

### 2-1. 탐지기
- **YOLO11**(CNN, PDF 충족) + **P2 헤드**(소형객체 전용 고해상 피처) + **입력 1280**
- 근거: CVPR2025 우승작도 YOLOv11x + AFPN + **P2 헤드**. 우리 데이터 38.6% small → P2/고해상 필수.
- 초장거리(수 픽셀)는 **SAHI 타일링**(오프라인/벤치 한정 — 실시간엔 무거움).

### 2-2. 시계열 (SOTA 방식 = 가벼운 게 이김)
- **Frame Dynamics**: `[현재프레임, 프레임차분, 광류(u,v)]`를 입력 채널로 concat
- **궤적 필터(TC-Filtering)**: 속도로 다음 위치 예측 → 창 밖 탐지 버림 (오탐↓) + **ByteTrack**
- 근거: CVPR2025 Best Paper "A Simple Detector with Frame Dynamics is a Strong Tracker" — **LSTM/Transformer 없이** 이김. 학습형 시계열(LSTM)은 보류(무겁고 Sim-to-Real 리스크).

### 2-3. Sim-to-Real 브리지 (핵심)
3개 레버 **동시**:
- **(a) DR(Domain Randomization)**: 배경·조명·드론외형·거리·방해물·카메라 무작위 → 실세계가 "또 하나의 변형"이 되게
- **(b) 센서 사실화**: 모션블러·노이즈·압축 (드론 빠름 + frame-dynamics 전이에 필수)
- **(c) 합성 사전학습 → 실데이터 소량 fine-tune** (PDF 방식). 실+합성 배치 혼합.
- 근거: NVIDIA/연구계 공통 — 포토리얼 단독도, DR 단독도 아닌 **하이브리드**가 최고.

### 2-4. 거리(10~500m) — 거리층화
- sim에서 거리 랜덤화로 10~500m 커버
- **자세 라벨 유효성 플래그**: 드론 픽셀 <임계(예 20px)면 자세 supervision **끔**(안 그러면 쓰레기 학습). 원거리는 탐지+궤적heading만.
- 평가도 **거리별로 분리**.

### 2-5. 오탐(드론 vs 새)
- **음성샘플**(드론0 프레임) + **방해물**(새·비행기)을 학습에 포함
- 시계열 궤적 일관성으로 깜빡임 오탐 억제

### 2-6. 평가 (Sim-to-Real 측정법)
- **공개벤치 절대기준**(Anti-UAV / DUT Anti-UAV) — 순환비교 지양(PDF)
- mAP + **거리별 분해** + 오탐분석 + 실데이터 라벨 IoU≥0.85 (PDF)

## 3. 동적 방법론 = 반복 갭-클로징 루프 ⭐
```
① 학습 → 실벤치 측정
② 실패 진단 (어떤 거리·배경·조건에서 미탐/오탐?)
③ AI2에 주문 (그 어려운 케이스를 더 생성 = 표적 DR)
④ 재학습 → 재측정 → ①
```
sim 무한생성 + 이 루프 = "막연히 많이"가 아니라 **부족분을 골라 채우는** 데이터중심 방법론.

## 4. 3-AI 분업 속 내 위치
```
나(1):  ①학습·측정 ②진단 → ③AI2에 주문서 → ④재학습 + 방법론 실험검증
AI2:    주문대로 sim 데이터 생성
AI3:    최신기법 자료 → 내가 검증·채택
```

## 5. 실험으로 검증할 가설 (GPU 비면 즉시)
| 가설 | 실험 | 기대 |
|---|---|---|
| 1280이 초소형 미탐을 줄인다 | `IMGSZ=1280` vs `640` 재학습, 거리별 recall 비교 | <32px recall↑ |
| 강증강 = DR 프록시, 일반화↑ | 증강 강도별 학습, hold-out 도메인 평가 | cross-domain mAP↑ |
| frame-dynamics가 오탐↓ | 광류/프레임차분 입력 추가(영상 필요) | FP↓·recall↑ |
| P2 헤드가 소형↑ | P2 on/off | small mAP↑ |

→ 결과 = AI2에 줄 "sim 생성 주문서"의 근거.
